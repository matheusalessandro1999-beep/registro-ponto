## Roadmap futuro (Firebase + Electron + ~50 usuários simultâneos)
O sistema será convertido para um aplicativo Desktop com Electron, backend Firebase, usado por ~50 usuários online fazendo alterações simultâneas.

### Ajustes necessários para o cenário multi-usuário:
1. **Concorrência** — Firebase Firestore com `onSnapshot` para atualizações em tempo real entre usuários; usar transações (`runTransaction`) para evitar sobrescrita em dados críticos (pesos, permissões, exclusões)
2. **Segurança no servidor** — Firebase Security Rules para garantir que permissões `canSee`/`canDo` sejam validadas no backend, não só no frontend
3. **Auditoria** — registrar `userId` + `timestamp` + `ip` em cada ação; já temos `registrarLog()` mas precisa ser expandido
4. **Loading states** — `withLoading()` já implementado para saves; precisa de equivalente para carregamentos iniciais (skeleton/spinner enquanto dados chegam do Firebase)
5. **Cache local + offline** — Firebase tem persistência offline nativa; o sistema atual usa localStorage como camada única — precisa adaptar para sincronizar com Firestore quando online
6. **Reatividade** — se 2 usuários estão na mesma página, alterações de um devem aparecer no outro automaticamente (Firebase `onSnapshot`)
7. **Importar/Exportar** — já existe, mas precisa incluir opção de backup no Firestore além do JSON local
8. **Electron build** — empacotar com Vite + electron-builder; CSS/JS separados em desenvolvimento viram bundle único minificado no .exe final
